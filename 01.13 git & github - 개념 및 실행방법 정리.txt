<Git>
* Git : 형상관리 프로그램(버전관리 도구)
    -> 안에 폴더를 형상관리 하는 기능
    -> 프로젝트 파일의 유지보수, 저장 용량, 코드 수정 이력 등의 전반적인 면에서 장점이 있음.

<GitHub>
* GitHub : Git으로 관리되는 폴더, 파일 등의 웹 호스팅 서비스 (aka. 구글 드라이버)
    -> 팀 프로젝트 소스코드를 GitHub에 공유하면 프로젝트 관리에 필요한 커뮤니티, 이슈 생성, 코드 리뷰 등 프로젝트 관리에 있어 다양한 기능을 제공

 
<사용자 지정 방법>
1. Git 실행
 - 바탕화면 혹은 특정 폴더 안에서 마우스 우측 클릭 -> 'open Git bash here'선택
 - vscode 터미널에서 Git 사용
 - Intellij 터미널에서 Git 사용
2. 사용자 지정(pc에서 최초 한번만 실행)
 - pc에 저장된 git 사용자 이메일 확인 : git config --global user.email
 - pc에 사용자의 이메일 추가 및 변경 : git config --global user.email 이메일주소
 - pc에 저장된 git 사용자의 name 확인 : git config --global user.name
 - pc에 저장된 git 사용자 name 추가 및 변경 : git config -- global user.name 사용자의 git username

<실행방법>
1. git으로 버전 관리가 필요한 폴더 생성

2. 해당 폴더 안에서 git bash 실행

3. 'git init' : 현재 폴더를 git 지역저장소(local repository)로 만들기
   -> 지역저장소가 되면 해당 폴더 안의 파일 및 폴더가 git에 의해 버전 관리 대상 후보가 됨.
   -> git init을 입력하면 .git 이라는 숨김 폴더가 생성
   -> .git 폴더가 생성된 폴더 안의 파일 및 폴더들은 git의 버전관리 대상이 됨.
   -> .git 폴더 안에는 버전 정보, 원격 저장소 주소 등이 저장

4. 현재 폴더의 임의의 텍스트 파일을 만들고 내용도 아무 내용이나 작성하고 저장

5. 'git status' 명령어를 입력하여 지역저장소의 상태 확인
   -> 지역저장소에 추가한 텍스트 파일이 untracked 상태로 표시
   -> untracked 상태 : 파일이 버전관리 대상이 되는 폴더에 존재하지만, 아직 버전관리를 시작하지 않은 상태를 의미

6. 'git add 파일명' : 버전관리 대상 파일로 지정
    ->파일이 commited 상태가 된 파일은 git의 버전관리 대상 파일이 됨 : staging 이라고 표현
    -> 'git add .' 명령어로 모든 파일을 일괄 staging 할 수 있음.  
    ex) .git가 존재하는  폴더 = 대형마트, untracked = 해당 폴더에 존재하는 모든 파일 및 폴더는 대형마트에 진열된 상품들, add = 카트에 물건 추가, staged = 진열된 상품 중 구매를 위해 카트에 옮겨담은 상품

7. 'git commit -m "커밋메세지" : 지금까지의 내용을 저장하는 명령어이자 복원시점을 만드는 명령어
    -> 자주 해주는 것이 좋음!
    -> 커밋메세지는 어떤 내용을 수정했는지 한눈에 알 수 있게 작성, 한글보단 영어로 작성하는 것이 나음.

8. github 사이트에서 원격저장소(remote repository)생성
    -> git으로 관리되는  폴더를 웹에 공유하기 위한 폴더를 만드는 것.

9. 'git remote add origin 원격저장소url' : 지역저장소와 원격저장소 연결
    -> origin은 원격저장소 이름

10. 'git branch -m main' : 원격 저장소와 지역 저장소의 브렌치 이름 통일시키기

11. 'git push origin main' : 지역저장소에서 작업한 내용을 원격저장소에 올리기

12. 이후 파일 추가 및 삭제, 코드 변경등의 작업 : add -> commit -> push 순으로 반복
