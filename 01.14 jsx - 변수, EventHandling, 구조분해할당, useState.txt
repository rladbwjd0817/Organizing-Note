<구조분해할당>
 : 구조를 분해해서 각 값을 할당한다.
 -> 배열, 객체에 저장된 다수의 데이터 각각을 편하게 사용하는 방법
 -> 구조분해할당을 사용하지 않으면 변수들이 많아져서 사용하는게 간결함.
    ex) const arr1 = [1, 7.5, 'hello'];
          const [a1, b1, c1] = arr1;
     => 변수 앞에 있는 [ ]는 배열이 아니라 구조분해할당임
     => a1 = 1, b1 = 7.5, c1 = 'hello'의 값이 저장

    ex) const arr2 = [5, 8.5, 'hello'];
          const [a2, b2] = arr2;
     => 배열의 수랑 변수 수 달라도 오류 안남!
     => a1 = 5, b1 = 8.5 값이 저장되고, 'hello'는 저장되지도, 출력되지도 않음
     => 배열의 수랑 변수 수가 다르면 맨 오른쪽 데이터는 저장 X , 출력 X
      
    ex) const arr3 = [5, 8.5, 'hello'];
         const [a3, b3, c3, d3] = arr3;
         console.log(d3); 
     => 이땐 undefined 출력

★★★<useState>★★★ - react에서만 있음
* useState : 2개의 데이터를 갖는 배열
* 0번째 자리 : 매개변수로 전달한 데이터 값
* 1번째 자리 : 함수
  => 1번째 함수 : 변수의 값을 변경하는 기능을 제공하는 함수

 ex) const[title, setTitile] = useState('Hi~');

★ 결론 ★
 - 데이터의 값 변경이 화면에 반영되려면 해당 데이터는 반드시 useState로 선언해야 함!!!!!!
 - state변수 : useState를 사용해 만들어진 변수 ex) title
 - state 변경 함수 : useState를 사용해 만들어진 함수 ex) setTitle
 - state 변경함수가 실행되면 컴포넌트를 리렌더링한다.
 - 리렌더링 : 컴포넌트의 처음부터 끝까지 다시 실행
 - state변수는 리렌더링 되더라도 초기화 되지 않는다!(다시 실행할 때 useState무시)

 ex) 
const UseState1 = () => {
  
  const [title, setTitle] = useState('hello~');
  console.log(5);

  return (
    <>
    <div>UseState1</div>
    <h2>{title}</h2>
    <button type='button' onClick={() => {
      setTitle(10); 
    }}>클릭</button>
    </>
  )
}

6. 10이 title에 들어가서 데이터가 바뀜
