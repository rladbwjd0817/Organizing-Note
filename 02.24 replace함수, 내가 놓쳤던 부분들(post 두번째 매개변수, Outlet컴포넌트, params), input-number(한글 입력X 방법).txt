  input type을 number로 주고 value 값을 주면 한글 입력 x
   ex) <input type="number" value={count} onChange={(e) => setCount(e.target.value)} />
	(1) type='number'의 특성 - 브라우저가 숫자만 허용하도록 필터링되고, 실제로는 빈 문자열이 value에 찍힘.
	(2) value로 제어되는 순간 더 엄격해져서 한글 씹힘
	* 한글 입력 시도
	 → e.target.value = "" (숫자가 아니니까)
	 → setCount("") 
	 → state = ""
	 → input의 value = "" 로 다시 렌더링
	 → 한글이 사라짐!
