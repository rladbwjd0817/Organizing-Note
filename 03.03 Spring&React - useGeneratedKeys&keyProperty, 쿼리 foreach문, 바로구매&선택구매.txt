<스프링>
1. 구매 등록 쿼리 작성
 -> 디비 테이블에 구매정보 테이블, 구매 상세 정보 테이블이 따로 있어서 insert 각각 테이블 두 개 작성
    => 리액트에서 받아야 할 값들이 buyPrice, memEmail, bookNum, buyCnt, buyNum 이라서 하나의 테이블에서 저 값들을 다 받을 수 없음 (컬럼이 없기 때문에)

2. buyNum컬럼이 SHOP_BUY테이블에서 기본키이자 BUY_DETAIL테이블에서 외래키로 적용하고 있어 두번째 insert문에서 buyNum 값을 useGeneratedKeys 속성을 활용해 넣어줄 것임
 -> useGeneratedKeys : auto_increment된 key를 insert시 사용하겠다는 의미
 -> keyProperty : useGeneratedKeys와 함께 사용되는 짝꿍! 이 쿼리의 빈 값을 채우기 위해 가져온 객체의 pk변수에 생성된 key를 넣어주겠다는 의미
  ex) <insert id="aa" useGeneratedKeys="true" keyProperty="buyNum">

3. 2번째 insert쿼리 문은 선택 구매 시 한 구매등록에 여러 상품들이 들어와야 하기에 반복문이 필요함!
 -> BuyDTO에서 구매 등록시 넘길 데이터의 관계를 봤을 때 1(buyDTO) : N(buyDetailDTO) 관계여서 BuyDTO클래스에 List<BuyDetailDTO> detailList; 리스트 변수 생성
 -> 리스트는 배열이기에 배열에 값을 넣으려면 반복문이 필요하고, 그 반복문을 실행하기 위해서는 <foreach>로 반복실행
 -> foreach : collection="반복돌릴 데이터(배열)" item="배열의 하나의 데이터" separator="반복되는 항목들 사이에 끼워넣는 문자열" 속성 작성
 -> <foreach>   반복 돌릴 쿼리   </foreach>
 -> 반복돌릴 쿼리에서 데이터를 꺼내올 때 어디에 저장되어 있는지 모르겠으면 리액트에서 console에 출력해봐

4. Mapper인터페이스 Buy, BuyDetail 각각 생성

5. BuyService 클래스 : 구매등록 정보 기능을 실행할 메서드 생성
 -> 어노테이션 @RequiredArgsConstructor은 멤버변수에 final 붙여야 해!!
 -> 바로 구매 & 선택 구매 동시에 만들거라 한 메서드 안에 기능 2개 넣기
 	=> 기능이 2개 설정되면 오류가 발생해도 에러 안나게 @Transactional(rollbackFor = Exception.class) 설정
	=> rollbackFor : 어떤 오류가 발생했을 때 롤백 시킬거야?
	=> Exception.class : 어떤 오류든 다 포함
	=> 어떤 오류든 다 롤백 시킬거야!!!
